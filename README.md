
## Подготовка проекта к запуску

Предполагается, что ***GIT*** и **_Node.js_** уже установлены на вашей машине глобально. Если нет, то скачиваем и выполняем установку.
Все команды необходимо набирать в консоли, находясь в корневой папке проекта.

Выполняем из консоли следующую команду (*точка в конце обязательна*):
```
> git clone https://github.com/gitolog/ss-toolkit.git .
```
При этом в корень проекта будут скопированы файлы **_gulpfile.js_** и **_package.json_**,необходимые для дальнейшей установки. 

### 1. Устанавливаем Gulp
Если Gulp уже установлен, этот пункт можно пропустить.
```
> npm install -g gulp
```
Ключ **-g** говорит о том, что устанавливается системно.

### 2. Устанавливаем необходимые модули Gulp

```
> npm install --save-dev
```

### 3. Работа с проектом. Запуск задач Gulp

Все команды набираются в окне терминала IDE WebStorm или PhpStorm

#### Общий BUILD всего тулкита
```
> gulp build
```
* Выполнится полная очистка папки **_/public_**

* Выполнится компиляция всех файлов **_*.styl_** из папки **_/src_**, превратив их в CSS и переложит в папку **_/public/css_**

* Выполнится минификация файлов **_*.css_** в папке **_/public/css_**

* Выполнится компиляция всех файлов **_*.js_** из папки **_/src_**. Скрипты будут объединены в один файл, который окажется в папке **_/public/js_**

* Выполнится минификация файлов **_*.js_** в папке **_/public/js_**

#### Задача быстрой компиляции (слежение за файлами)
```
> gulp
```
Эта задача запускается для отладки при внесении изменений в исходные файлы стилей и скриптов тулкита. Результат компиляции будет перезаписывать выходные файлы в папке **_/public_**

* При изменении любого из файлов **_*.styl_** в папке **_/src_** выполнится сборка CSS, но без минификации

* При изменении любого из файлов **_*.js_** в папке **_/src_** выполнится сборка JS, но без минификации
